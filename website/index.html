AI가 작업 끝날 때마다 핑.
한국 개발자를 위한 사운드 알림.

▾ 30초 설치 (macOS / Linux / WSL2)
curl -fsSL https://cromaizing.com/cromaiping/install.sh | bash

이런 적 있으신가요?

  • AI한테 작업 시키고 다른 일 하다가 20분 만에 돌아왔는데 5분 전에 끝나있던 적
  • 권한 승인 기다리는데 모니터 안 보고 있어서 계속 멈춰있던 것을 몰랐던 적
  • AI가 에러 토했는데 모르고 다른 거 시키던 적
  • 회의 중인데 동료가 "AI 응답 왔어?" 물어볼 때마다 화면 봐야 했던 적

크로마이핑이 해결합니다. 설치하면 바로 작동, 설정 필요 없음.

주요 기능

🔔

7가지 이벤트 인식

세션 시작, 작업 완료, 오류, 권한 대기, 컨텍스트 한계, 빠른 입력 자동 감지

한국어 슬래시 명령어

채팅에서 /크로마이핑-상태 같은 한국어 명령 바로 작동

🔐

완전 로컬

클라우드 전송 0, 텔레메트리 0. ~/.claude/ 안에서만 동작.

⚡

경량

약 1200줄 bash + Python. 외부 의존성 최소.

🎯

경로별 다른 팩

회사 프로젝트는 차분한 팩, 개인 프로젝트는 재밌는 팩으로 자동 전환

주요 명령어

설치 후 어디서든 사용 가능:

cromaiping status              # 현재 상태
cromaiping toggle              # 음소거 토글
cromaiping volume 70           # 볼륨 변경
cromaiping list                # 설치된 팩 목록
cromaiping preview             # 사운드 미리듣기
cromaiping use <팩이름>        # 팩 변경

Claude Code 채팅에서 슬래시 명령어로:

/cromaiping-status             현재 상태
/cromaiping-toggle             음소거 토글
/cromaiping-volume 70          볼륨 변경
/cromaiping-use <팩>           팩 변경
/크로마이핑-상태               한국어도 OK

자주 묻는 질문

peon-ping과 뭐가 달라요?

peon-ping의 한국 시장 특화 버전입니다. 같은 CESP 표준을 쓰기 때문에 외부 사운드팩은 호환됩니다. 차이는 한국어 UI/문서, 한국 특화 기능 (카카오톡 알림, 한국어 TTS — v1.0 예정), cromaizing.com 생태계 통합입니다.

둘 다 깔아도 돼요?

됩니다. 같은 이벤트에 둘 다 반응하니 사운드가 2번 울려요. 한쪽 음소거 추천: peon pause 또는 cromaiping pause

사운드를 직접 만들어도 되나요?

네! CESP v1.0 표준대로 openpeon.json + sounds/ 폴더만 있으면 됩니다.

회사에서 써도 되나요?

MIT 라이선스라 상업적 사용 가능합니다. 단, 사용하시는 사운드팩의 라이선스(CC-BY-NC-4.0 등)는 별도로 확인하세요.

회의 중에 사운드 안 울리게 할 수 있나요?

v0.2부터 자동 회의 감지 (마이크 사용 중 감지) 추가 예정입니다. 지금은 cromaiping pause로 수동 토글.
