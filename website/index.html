Why Cromaiping

이런 적 한 번쯤 있죠?

AI 코딩 도구는 좋은데, 작업 끝났는지 매번 확인하기 귀찮으시죠.

⏰

AI한테 작업 시키고 다른 일 하다가 20분 만에 돌아왔는데, 알고 보니 5분 전에 끝나있던 적

🚫

권한 승인 기다리는데 모니터 안 보고 있어서 계속 멈춰있던 것을 모르고 지나간 적

😱

AI가 에러 토했는데 모르고 다음 작업 계속 시키던 적

👀

회의 중인데 동료가 "AI 응답 왔어?" 물어볼 때마다 화면 봐야 했던 적

시작하기

3단계로 끝나는 설치

복사 → 붙여넣기 → 끝. 추가 설정 없이 바로 작동합니다.

1

터미널을 열어주세요

맥은 ⌘ + Space 누르고 "터미널" 검색. 또는 iTerm, Warp, VS Code 터미널 등 아무거나 OK.

💡 처음이세요? 터미널은 그냥 글자로 명령을 내리는 검은 창이에요. 무서운 거 아닙니다.
2

아래 한 줄을 복사해서 붙여넣고 엔터

다운로드, 설치, Claude Code 훅 등록까지 다 자동으로 해줘요.

curl -fsSL https://cromaizing.com/cromaiping/install.sh | bash
💡 뭐가 일어나는지 알고 싶다면? curl -fsSL https://cromaizing.com/cromaiping/install.sh -o install.sh 로 파일만 받아서 내용 먼저 보실 수 있어요.
3

Claude Code를 재시작하세요

설치 시 ~/.claude/settings.json 에 훅이 자동 등록됩니다. 재시작하면 새 설정이 적용돼요.

✅ 끝! 이제 Claude Code가 작업 끝내면 사운드가 울립니다. cromaiping preview 로 미리 들어보실 수 있어요.
CLI 명령어

터미널에서 사용하기

설치 후 어디서든 cromaiping 명령으로 제어할 수 있어요.

cromaiping status
현재 상태 확인 (활성/음소거, 기본 팩, 볼륨, 알림 설정) cromaiping status
cromaiping toggle
음소거 ↔ 활성 토글. 회의나 통화 중에 빠르게 끄기 좋아요. cromaiping toggle
cromaiping volume <0-100>
볼륨 변경. 0은 무음, 100은 최대. 인자 없으면 현재 볼륨 표시. cromaiping volume 70
cromaiping list
설치된 사운드 팩 목록 보기. cromaiping list
cromaiping use <팩이름>
기본 팩 변경. 모든 새 세션에 적용돼요. cromaiping use cromaiping_default
cromaiping preview [카테고리]
사운드 미리듣기. 카테고리 안 적으면 session.start 재생. cromaiping preview task.complete
cromaiping help
전체 명령어 목록 보기. cromaiping help

Events

어떤 상황에 어떤 사운드?

크로마이핑은 7가지 상황을 자동으로 인식합니다.

상황
카테고리
🎬세션 시작 / Claude Code 실행
session.start
✅작업 완료 (Stop 이벤트)
task.complete
🛑오류 발생 (Bash 실패 등)
task.error
🔐권한 승인 대기
input.required
⚠️컨텍스트 한계 (PreCompact)
resource.limit
🤬너무 빠른 입력 (10초 내 3+회)
user.spam
👋작업 수락 (옵션, 기본 OFF)
task.acknowledge
💡 각 카테고리는 ~/.claude/hooks/cromaiping/config.json 에서 끌 수 있습니다.

FAQ

자주 묻는 질문

peon-ping과 뭐가 달라요?

peon-ping의 한국 시장 특화 버전이라고 보시면 됩니다. 같은 CESP 표준을 쓰기 때문에 외부 사운드팩은 호환됩니다.

차이점:

  • 한국어 UI · 문서 · 슬래시 명령어 (/크로마이핑-상태 등)
  • 한국 개발자 환경 기준 트러블슈팅 가이드
  • v1.0에 카카오톡 알림, 한국어 TTS, cromaizing.com 통합 예정
peon-ping 이미 쓰는데 둘 다 깔아도 돼요?

됩니다. 둘 다 같은 이벤트에 반응하니 사운드가 두 번 울려요. 한쪽 음소거하시는 게 깔끔합니다:

peon pause 또는 cromaiping pause

회의 중에 사운드 안 울리게 할 수 있나요?

지금은 cromaiping pause / cromaiping resume으로 수동 토글하셔야 해요.

v0.2부터 자동 회의 감지가 추가됩니다 (마이크 사용 중일 때 자동 음소거). 또는 채팅에서 /크로마이핑-음소거 한 번 치면 바로 꺼집니다.

사운드를 직접 만들어도 되나요?

네! CESP v1.0 표준대로 openpeon.json 매니페스트와 sounds/ 폴더만 있으면 됩니다.

만든 팩은 ~/.claude/hooks/cromaiping/packs/<팩이름>/ 에 넣으면 자동 인식. CESP 팩 만들기 가이드 →

회사에서 써도 되나요? 라이선스가 어떻게 되나요?

크로마이핑 본체는 MIT 라이선스라 상업적 사용 자유롭습니다.

단, 사용하는 사운드팩은 별도 라이선스를 가질 수 있어요 (CC-BY-NC-4.0 등). 외부 팩 다운로드 시 각 팩의 LICENSE를 확인해주세요.

제거하려면 어떻게 하나요?

설치된 디렉토리에서 uninstall.sh를 실행하시면 됩니다:

bash ~/.claude/hooks/cromaiping/uninstall.sh

훅 등록도 같이 해제되고, 다운로드한 사운드팩은 백업할지 물어봐요.

Linux나 Windows에서도 동작해요?

macOS: v0.1부터 완전 지원 ✅
Linux: v0.1부터 기본 지원 (PipeWire/PulseAudio/ffplay 자동 감지) ✅
WSL2: v0.1부터 PowerShell MediaPlayer로 동작 ✅
Windows 네이티브: v0.3 예정 🚧

인터넷 없는 환경에서도 동작하나요?

네. 설치 후엔 완전 오프라인 동작합니다. 사운드 재생, 알림, 슬래시 명령어 전부 로컬에서만 돌아가요. 외부 서버 호출 0회입니다.
